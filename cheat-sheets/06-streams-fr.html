Notions essentielles

Un Stream<T> est une séquence d'éléments sur laquelle on peut enchaîner des opérations de traitement sans modifier la source. Il n'est pas une structure de données : il ne stocke rien et ne peut être consommé qu'une seule fois.

Un pipeline Stream se compose de trois parties :

  • Source : la collection, le tableau ou la méthode qui produit les éléments.
  • Opérations intermédiaires : transformations paresseuses qui retournent un nouveau Stream (filter, map, flatMap, distinct, sorted, …).
  • Opération terminale : déclenche réellement l'exécution et consomme le Stream (collect, count, max, forEach, …).
Un Stream ne peut être consommé qu'une seule fois. Appeler deux opérations terminales sur le même Stream lance une IllegalStateException.
Type de Stream Classe Remarque
Objets Stream<T> Cas général.
Entiers primitifs IntStream Évite le boxing/unboxing des int.
Longs primitifs LongStream Évite le boxing/unboxing des long.
Doubles primitifs DoubleStream Évite le boxing/unboxing des double.

La méthode stream() est disponible directement sur toute Collection (comme List ou Set) sans import supplémentaire.

Créer un Stream

Depuis une collection

Collection<T>.stream() Collection<T>.parallelStream() 
List<String> noms = List.of("Alice", "Bob", "Claire");

Stream<String> stream = noms.stream();

C'est la manière la plus courante. On part d'une List et on enchaîne les opérations.

Depuis des valeurs littérales

Stream.of(T... values) 
Stream<String> stream = Stream.of("Alice", "Bob", "Claire");

Depuis un tableau

Arrays.stream(T[] array) 
String[] tableau = {"Alice", "Bob"};
Stream<String> stream = Arrays.stream(tableau);

Stream vide

Stream.empty() 
Stream<String> vide = Stream.empty();

Opérations intermédiaires

Les opérations intermédiaires sont paresseuses : elles ne s'exécutent pas tant qu'aucune opération terminale n'est appelée. Chacune retourne un nouveau Stream.

filter

Stream<T> filter(Predicate<? super T> predicate)

Retient uniquement les éléments pour lesquels le prédicat est vrai.

List<Produit> cheers = produits.stream()
    .filter(p -> p.getPrix() > 100)
    .collect(Collectors.toList());

List<Produit> electronique = produits.stream()
    .filter(p -> p.getCategorie().equals("Electronique"))
    .collect(Collectors.toList());

map

Stream<R> map(Function<? super T, ? extends R> mapper)

Transforme chaque élément en un autre élément. Le type de sortie peut être différent du type d'entrée.

// Extraire le prix de chaque produit
List<Integer> prix = produits.stream()
    .map(Produit::getPrix)
    .collect(Collectors.toList());

// Extraire les noms des produits
List<String> noms = produits.stream()
    .map(Produit::getNom)
    .collect(Collectors.toList());
La syntaxe Produit::getNom est une référence de méthode. Elle est équivalente à la lambda p -> p.getNom().

mapToInt, mapToLong, mapToDouble

IntStream mapToInt(ToIntFunction<? super T> mapper)

Transforme un Stream<T> en stream primitif. Utile pour des calculs numériques (sum, average, max) sans boxing.

List<Integer> notes = List.of(12, 8, 15, 9, 18);
int somme = notes.stream()
    .mapToInt(Integer::intValue)
    .sum();

flatMap

Stream<R> flatMap(Function<? super T, ? extends Stream<? extends R>> mapper)

"Aplatit" une structure imbriquée. Pour chaque élément, la fonction retourne un Stream d'éléments ; tous ces streams sont concaténés en un seul stream.

List<List<String>> groupes = List.of(
    List.of("alpha", "beta"),
    List.of("gamma", "delta")
);
List<String> mots = groupes.stream()
    .flatMap(List::stream)
    .collect(Collectors.toList());
// ["alpha", "beta", "gamma", "delta"]
flatMap est la solution pour "passer d'une liste de listes à une liste plate". Si chaque élément contient une List, appeler .stream() sur cette liste à l'intérieur du flatMap.

distinct

Stream<T> distinct()

Supprime les doublons en utilisant equals et hashCode. L'ordre d'apparition des éléments est préservé.

List<String> mots = List.of("chat", "chien", "chat", "oiseau");
List<String> uniques = mots.stream()
    .distinct()
    .collect(Collectors.toList());
// ["chat", "chien", "oiseau"]

sorted

Stream<T> sorted() Stream<T> sorted(Comparator<? super T> comparator)

Trie les éléments selon l'ordre naturel ou un comparateur explicite.

// Trier les noms de produits par ordre alphabétique
List<String> nomsTries = produits.stream()
    .map(Produit::getNom)
    .sorted()
    .collect(Collectors.toList());

// Trier les produits par prix croissant
List<Produit> tries = produits.stream()
    .sorted(Comparator.comparingInt(Produit::getPrix))
    .collect(Collectors.toList());

limit et skip

Stream<T> limit(long maxSize) Stream<T> skip(long n)

limit tronque le stream aux maxSize premiers éléments. skip ignore les n premiers éléments.

// Les 3 premiers produits
List<Produit> premiers = produits.stream()
    .limit(3)
    .collect(Collectors.toList());

peek

Stream<T> peek(Consumer<? super T> action)

Applique une action sur chaque élément sans les modifier, utile pour déboguer un pipeline.

produits.stream()
    .peek(p -> System.out.println("Produit : " + p.getNom()))
    .filter(p -> p.getCategorie().equals("Electronique"))
    .collect(Collectors.toList());

Opérations terminales

Une opération terminale déclenche le traitement du pipeline et consomme le Stream. Après son appel, le Stream est fermé.

collect

R collect(Collector<? super T, A, R> collector)

Accumule les éléments dans un conteneur. C'est l'opération terminale la plus courante. Les Collector prêts à l'emploi sont dans la classe Collectors.

List<Produit> electronique = produits.stream()
    .filter(p -> p.getCategorie().equals("Electronique"))
    .collect(Collectors.toList());

count

long count()

Retourne le nombre d'éléments dans le stream.

long nombreElectronique = produits.stream()
    .filter(p -> p.getCategorie().equals("Electronique"))
    .count();

max et min

Optional<T> max(Comparator<? super T> comparator) Optional<T> min(Comparator<? super T> comparator)

Retourne l'élément maximal ou minimal selon un comparateur. Retourne un Optional vide si le stream est vide.

List<Integer> nombres = List.of(3, 1, 4, 1, 5, 9, 2, 6);

Optional<Integer> max = nombres.stream()
    .max(Comparator.naturalOrder());

OptionalInt max2 = nombres.stream()
    .mapToInt(Integer::intValue)
    .max();

findFirst et findAny

Optional<T> findFirst() Optional<T> findAny()

Retourne un Optional contenant le premier élément (findFirst) ou n'importe quel élément (findAny). Retourne un Optional vide si le stream est vide.

Optional<Produit> premierCher = produits.stream()
    .filter(p -> p.getPrix() > 100)
    .findFirst();

anyMatch, allMatch, noneMatch

boolean anyMatch(Predicate<? super T> predicate) boolean allMatch(Predicate<? super T> predicate) boolean noneMatch(Predicate<? super T> predicate)

Vérifient si au moins un, tous, ou aucun des éléments satisfont le prédicat.

boolean aUnCherProduit = produits.stream()
    .anyMatch(p -> p.getPrix() > 100);

boolean tousAvecPrix = produits.stream()
    .allMatch(p -> p.getPrix() > 0);

forEach

void forEach(Consumer<? super T> action)

Applique une action sur chaque élément. Ne retourne rien.

produits.stream()
    .forEach(p -> System.out.println(p.getNom()));
forEach ne permet pas de construire un résultat. Pour collecter ou transformer, utiliser collect.

reduce

Optional<T> reduce(BinaryOperator<T> accumulator) T reduce(T identity, BinaryOperator<T> accumulator)

Combine les éléments deux à deux pour produire un seul résultat.

// Somme des prix (avec valeur initiale)
int total = produits.stream()
    .mapToInt(Produit::getPrix)
    .reduce(0, Integer::sum);

Collectors

Classe : java.util.stream.Collectors. Fournit des implémentations prêtes à l'emploi de l'interface Collector.

toList

Collector toList()

Collecte les éléments dans une List. C'est le collector le plus utilisé.

List<String> noms = produits.stream()
    .map(Produit::getNom)
    .collect(Collectors.toList());

toSet

Collector toSet()

Collecte les éléments dans un Set. Les doublons sont éliminés automatiquement. L'ordre n'est pas garanti.

Set<String> categoriesUniques = produits.stream()
    .map(Produit::getCategorie)
    .collect(Collectors.toSet());

toMap

Collector toMap(Function keyMapper, Function valueMapper) Collector toMap(Function keyMapper, Function valueMapper, BinaryOperator mergeFunction)

Collecte les éléments dans une Map. Lance une exception si deux éléments produisent la même clé, sauf si une fonction de fusion est fournie.

// Associer le nom d'un produit à son prix
Map<String, Integer> prixParNom = produits.stream()
    .collect(Collectors.toMap(Produit::getNom, Produit::getPrix));

groupingBy

Collector groupingBy(Function classifier) Collector groupingBy(Function classifier, Collector downstream)

Regroupe les éléments selon un critère. Retourne une Map<K, List<T>> par défaut. Un collector aval (downstream) peut transformer la valeur de chaque groupe.

List<String> mots = List.of("chat", "chien", "cheval", "aigle", "ara");

// Grouper par longueur du mot
Map<Integer, List<String>> parLongueur = mots.stream()
    .collect(Collectors.groupingBy(String::length));

// Compter le nombre de mots par longueur
Map<Integer, Long> compteParLongueur = mots.stream()
    .collect(Collectors.groupingBy(
        String::length,
        Collectors.counting()
    ));

counting

Collector counting()

Compte le nombre d'éléments. Souvent utilisé comme collector aval de groupingBy.

List<String> mots = List.of("chat", "chien", "cheval", "aigle", "ara");
Map<Integer, Long> compteParLongueur = mots.stream()
    .collect(Collectors.groupingBy(
        String::length,
        Collectors.counting()
    ));

joining

Collector joining() Collector joining(CharSequence delimiter) Collector joining(CharSequence delimiter, CharSequence prefix, CharSequence suffix)

Concatène les éléments String en une seule chaîne.

String nomsJoints = produits.stream()
    .map(Produit::getNom)
    .collect(Collectors.joining(", "));
// "ProA, ProB, ProC"

summarizingInt / averagingInt / summingInt

Collector summingInt(ToIntFunction mapper) Collector averagingInt(ToIntFunction mapper) Collector summarizingInt(ToIntFunction mapper)

Collecteurs numériques. summingInt calcule une somme, averagingInt une moyenne, summarizingInt retourne un IntSummaryStatistics contenant count, sum, min, max et average.

List<Integer> nombres = List.of(3, 1, 4, 1, 5, 9);
int somme = nombres.stream()
    .collect(Collectors.summingInt(Integer::intValue));

Optional

Classe : java.util.Optional<T>. Un conteneur pouvant contenir une valeur ou être vide. Il permet d'éviter les retours null et les NullPointerException associées. Les opérations terminales max, min, findFirst et findAny retournent un Optional.

Créer un Optional

static <T> Optional<T> of(T value) static <T> Optional<T> ofNullable(T value) static <T> Optional<T> empty() 
Optional<String> present = Optional.of("Alice");
Optional<String> peut_etre = Optional.ofNullable(valeurPotentiellementNulle);
Optional<String> vide = Optional.empty();

Optional.of(null) lance une NullPointerException. Utiliser ofNullable si la valeur peut être null.

Vérifier et lire la valeur

boolean isPresent() boolean isEmpty() T get() 
Optional<Integer> prixMax = produits.stream()
    .map(Produit::getPrix)
    .max(Comparator.naturalOrder());

if (prixMax.isPresent()) {
    System.out.println("Maximum : " + prixMax.get());
}
Appeler get() sur un Optional vide lance une NoSuchElementException. Toujours vérifier avec isPresent() avant d'appeler get(), ou préférer les méthodes ci-dessous.

Obtenir une valeur par défaut

T orElse(T other) T orElseGet(Supplier<? extends T> supplier) <X extends Throwable> T orElseThrow(Supplier<? extends X> exceptionSupplier) 
int max = produits.stream()
    .mapToInt(Produit::getPrix)
    .max()
    .orElse(0);

Optional<String> optNom = Optional.ofNullable(produit.getNom());
String nom = optNom.orElse("Inconnu");
String nom2 = optNom.orElseGet(() -> "valeur-par-defaut");

Transformer un Optional

Optional<U> map(Function<? super T, ? extends U> mapper) Optional<U> flatMap(Function<? super T, ? extends Optional<? extends U>> mapper) Optional<T> filter(Predicate<? super T> predicate) 
Optional<Produit> optProduit = Optional.of(new Produit("Stylo", "Bureau", 2));
Optional<String> nom = optProduit
    .filter(p -> p.getPrix() > 10)
    .map(Produit::getNom);

Exécuter du code si valeur présente

void ifPresent(Consumer<? super T> action) void ifPresentOrElse(Consumer<? super T> action, Runnable emptyAction) 
prixMax.ifPresent(v -> System.out.println("Prix max : " + v));

Exemples complets

Pipeline sur une liste de chaînes

List<String> villes = List.of("Paris", "Lyon", "Pau", "Périgueux", "Laon");

// Villes dont le nom a plus de 4 lettres, triées
List<String> longues = villes.stream()
    .filter(v -> v.length() > 4)
    .sorted()
    .collect(Collectors.toList());
// ["Lyon", "Paris", "Périgueux"]

Pipeline sur une liste d'entiers

List<Integer> notes = List.of(12, 8, 15, 12, 9, 15, 18);

// Valeurs distinctes, triées
List<Integer> distinctes = notes.stream()
    .distinct()
    .sorted()
    .collect(Collectors.toList());
// [8, 9, 12, 15, 18]

// Maximum
Optional<Integer> max = notes.stream()
    .max(Comparator.naturalOrder());

// Somme
int somme = notes.stream()
    .mapToInt(Integer::intValue)
    .sum();

flatMap sur une liste de listes

List<List<String>> groupes = List.of(
    List.of("alpha", "beta", "alpha"),
    List.of("gamma", "beta")
);

List<String> tous = groupes.stream()
    .flatMap(List::stream)
    .collect(Collectors.toList());
// ["alpha", "beta", "alpha", "gamma", "beta"]

groupingBy + counting sur des chaînes

List<String> mots = List.of("chat", "chien", "cheval", "aigle", "ara");

Map<Integer, Long> parLongueur = mots.stream()
    .collect(Collectors.groupingBy(
        String::length,
        Collectors.counting()
    ));
// {3=1, 4=1, 5=2, 6=1}

Pièges fréquents

Problème Cause probable Correction
Le stream ne produit aucun résultat alors que la source n'est pas vide. L'opération terminale n'a pas été appelée ; les opérations intermédiaires seules ne s'exécutent pas. Ajouter une opération terminale comme collect, count, forEach.
IllegalStateException: stream has already been operated upon or closed Le stream a déjà été consommé par une opération terminale. Appeler de nouveau .stream() sur la source pour créer un nouveau stream.
NullPointerException dans un map ou filter. Un élément de la source est null ou un getter retourne null. Filtrer les null avec filter(Objects::nonNull) avant la transformation.
Utiliser map au lieu de flatMap avec une liste imbriquée. map(g -> g.getElements()) retourne un Stream<List<T>>, pas un Stream<T>. Utiliser flatMap(g -> g.getElements().stream()) pour aplatir.
Appel à Optional.get() sur un Optional vide. Le stream était vide, donc max, min ou findFirst ont retourné un Optional vide. Utiliser orElse(...), orElseGet(...) ou vérifier avec isPresent() avant get().
Doublons non supprimés malgré l'utilisation de distinct(). La classe des éléments ne redéfinit pas equals et hashCode. Redéfinir equals et hashCode dans la classe, ou mapper vers un type qui les implémente (ex. String) avant distinct().
Confondre map et mapToInt. map retourne un Stream<Integer> (objets boxés). mapToInt retourne un IntStream (primitifs) avec des méthodes supplémentaires comme sum() et average(). Utiliser mapToInt quand on a besoin de sum(), average() ou max() sans comparateur.
toMap lance une exception si deux éléments ont la même clé. Clés dupliquées sans fonction de fusion. Ajouter une fonction de fusion : Collectors.toMap(keyFn, valueFn, (v1, v2) -> v1).
