Fichiers .properties

Format texte simple : une paire clé=valeur par ligne, # pour les commentaires. Il externalise la configuration hors du code : on change le comportement sans recompiler.

# blacklist.properties
blacklist.actif=true
blacklist.service=blacklist.BlacklistServiceImpl
blacklist.urls=example.org,test.example.com
Convention : le fichier vit à la racine du projet ou dans les ressources (src/main/resources pour un projet Maven, embarqué dans le jar).

API java.util.Properties

Properties étend Hashtable<Object,Object> ; en pratique on ne manipule que des String.

void load(InputStream inStream) throws IOException void load(Reader reader) throws IOException String getProperty(String key) String getProperty(String key, String defaultValue) Object setProperty(String key, String value) void store(Writer writer, String comments) throws IOException Set<String> stringPropertyNames() 
Properties props = new Properties();

// Depuis un fichier sur disque
try (InputStream in = new FileInputStream("blacklist.properties")) {
    props.load(in);
}

// Ou depuis les ressources du classpath (fonctionne aussi dans un jar)
try (InputStream in = Main.class.getResourceAsStream("/blacklist.properties")) {
    props.load(in);
}

boolean actif = Boolean.parseBoolean(props.getProperty("blacklist.actif", "false"));
String[] urls = props.getProperty("blacklist.urls", "").split(",");
getProperty renvoie toujours une String (ou null) : convertir explicitement (Boolean.parseBoolean, Integer.parseInt…) et prévoir une valeur par défaut.

DI configurée par properties

Combinaison des semaines précédentes : le fichier properties contient le nom de la classe à instancier, la factory la crée par introspection, le client la reçoit par injection. Changer d'implémentation = éditer le fichier, zéro recompilation.

String nomClasse = props.getProperty("blacklist.service");

BlacklistService service = (BlacklistService) Class.forName(nomClasse)
        .getDeclaredConstructor()
        .newInstance();

ProxyServer serveur = new ProxyServer(service);   // injection classique

Packaging : jar exécutable avec Maven

Un jar (Java ARchive) regroupe classes compilées et ressources. Pour le rendre exécutable, son manifest doit indiquer la classe principale.

Configuration du pom.xml

<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-jar-plugin</artifactId>
      <configuration>
        <archive>
          <manifest>
            <mainClass>main.Main</mainClass>
          </manifest>
        </archive>
      </configuration>
    </plugin>
  </plugins>
</build>

Commandes

mvn package        # compile, teste et produit target/monprojet-1.0.jar
java -jar target/monprojet-1.0.jar        # exécute la mainClass du manifest
java -cp target/monprojet-1.0.jar autre.Main   # exécute une autre classe
  • mvn package enchaîne compile → test → jar ; un test rouge bloque le packaging.
  • Les fichiers de src/main/resources sont embarqués dans le jar — les lire avec getResourceAsStream, pas avec un chemin disque.

Javadoc - rédiger

Commentaire /** ... */ juste au-dessus de la classe, méthode ou attribut documenté. Première phrase = résumé (elle apparaît dans les index générés).

Tag Rôle
@param nom description Décrit un paramètre (un tag par paramètre).
@return description Décrit la valeur de retour (jamais sur un void).
@throws Exception description Décrit quand l'exception est lancée.
@see référence Renvoi vers une autre classe ou méthode.
@deprecated raison Marque un élément obsolète et indique le remplacement.
{@link Classe#methode} Lien cliquable dans le texte de la description.
{@code expression} Formatage code dans la description. 
/**
 * Vérifie si une URL est présente dans la liste noire.
 *
 * @param url l'URL à vérifier, non nulle
 * @return {@code true} si l'URL est bloquée, {@code false} sinon
 * @throws IllegalArgumentException si {@code url} est nulle ou vide
 * @see #ajouter(String)
 */
boolean estBloquee(String url);
Documenter le contrat (quoi, conditions, exceptions), pas l'implémentation (comment). La Javadoc s'écrit sur l'interface d'abord : les implémentations en héritent dans la documentation générée.

Javadoc - générer

Depuis IntelliJ

  1. Tools → Generate JavaDoc…
  2. Choisir la portée (projet ou package) et le dossier de sortie (par ex. doc/).
  3. Locale : fr_BE ou fr_FR pour des libellés français.
  4. Ouvrir doc/index.html dans un navigateur.

En ligne de commande

javadoc -d doc -sourcepath src/main/java -subpackages domaine:server

# ou via Maven
mvn javadoc:javadoc     # sortie dans target/site/apidocs

La documentation générée reflète la visibilité : par défaut, seuls les membres public et protected apparaissent.

Exemple complet neutre

import java.io.FileInputStream;
import java.io.IOException;
import java.io.InputStream;
import java.util.Properties;

/**
 * Charge la configuration de l'application depuis un fichier properties.
 */
public class Configuration {

    private final Properties props = new Properties();

    /**
     * Charge le fichier de configuration donné.
     *
     * @param chemin le chemin du fichier .properties, non nul
     * @throws IOException si le fichier est introuvable ou illisible
     */
    public Configuration(String chemin) throws IOException {
        try (InputStream in = new FileInputStream(chemin)) {
            props.load(in);
        }
    }

    /**
     * Renvoie la valeur entière associée à la clé.
     *
     * @param cle la clé recherchée
     * @param defaut la valeur renvoyée si la clé est absente
     * @return la valeur entière de la clé, ou {@code defaut} si absente
     */
    public int getEntier(String cle, int defaut) {
        String valeur = props.getProperty(cle);
        return valeur == null ? defaut : Integer.parseInt(valeur);
    }
}

Pièges fréquents

Problème Cause probable Correction
getProperty renvoie null Clé mal orthographiée ou fichier non chargé. Vérifier la clé exacte ; fournir une valeur par défaut.
Fichier properties introuvable une fois dans le jar Lecture par chemin disque (FileInputStream). Le placer dans les ressources et lire via getResourceAsStream.
no main manifest attribute au lancement du jar mainClass absent du manifest. Configurer maven-jar-plugin avec la mainClass.
ClassNotFoundException au chargement configuré Nom de classe incomplet dans le properties. Utiliser le nom pleinement qualifié (package.Classe).
Javadoc générée vide ou incomplète Membres package-private, ou commentaires // au lieu de /** */. Documenter les éléments publics avec de vrais commentaires Javadoc.
Avertissements javadoc au build @param/@return manquants ou paramètres renommés. Un @param par paramètre, noms exacts, @return sauf void.
